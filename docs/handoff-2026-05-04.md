## What Shipped

### Provider and Data Quality

- Codex ingestion now shares the existing normalized session model instead of a
  parallel product path.
- Dashboard, session detail, autopsy, and aggregate queries are provider-aware.
- Redaction, redacted backups, and redaction audit commands are available for
  safer fixture and backup workflows.
- The Insights page includes an Investigation Queue for sessions with review
  signals such as permission friction, weak attribution, file-access failures,
  no-edit expensive work, suspiciously low active time, and residual error
  categories.

### Semantic Artifacts

- Added semantic artifact tables and models:
  `decision`, `setup_step`, `credential_step`, `verification_recipe`,
  `agent_mistake`, `risky_action`, `future_agent_instruction`, and
  `planner_signal`.
- Added artifact persistence helpers for proposal, evidence, lifecycle events,
  accept, and reject.
- Added heuristic `aide digest <session-id>` proposals with
  `--save-proposals`.
- Added CLI review commands:
  `aide artifacts list`, `show`, `accept`, and `reject`.
- Added `/artifacts` web review UI with filters and accept/reject actions.

### Generated Context

- Added deterministic runbook generation from accepted artifacts:
  `aide runbook generate --project <name>`.
- Added deterministic task brief generation from accepted artifacts:
  `aide brief --project <name> --task "<task>"`.
- Added `/runbook` web preview dashboard.
- Added `/brief` web preview dashboard with project selector and task input.

### Security Maintenance

- Updated Flask and Werkzeug for Dependabot CVEs.
